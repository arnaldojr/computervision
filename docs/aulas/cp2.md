# Aula 8 - CP2

## Objetivo

Avaliação do projeto entregue na CP2, com foco em arquitetura, funcionalidade e organização do código.

## Critérios de Avaliação

Durante esta aula, serão avaliados os seguintes aspectos do projeto entregue:

### Arquitetura (30%)
- Separação adequada de responsabilidades (Controller, Service, Model)
- Estrutura de diretórios organizada
- Uso apropriado de padrões de projeto

### Funcionalidade (30%)
- API funcional e respondendo corretamente às requisições
- Classificação de imagens funcionando adequadamente
- Tratamento adequado de erros

### Organização do Código (20%)
- Código limpo e bem comentado
- Nomenclatura consistente
- Seguimento de convenções de estilo

### Documentação (20%)
- README com instruções claras
- Documentação dos endpoints
- Explicação do funcionamento do sistema

## Atividade

Durante esta aula, os alunos deverão:

1. Apresentar sua API de classificação
2. Demonstrar o funcionamento dos endpoints
3. Explicar a arquitetura escolhida
4. Discutir desafios enfrentados e soluções implementadas
